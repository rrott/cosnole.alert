cosnole.alert

Browser Extension to shows logs, errors and warnings as toast messages to help you catch bugs faster.
It extends browser's console object with the alert() function.
Ok
With "cosnole.alert" you can:
  • reflect console.log() calls in the toast messages
  • trigger window.alert() on all console.log() calls
  • use console.alert() instead of window.alert()
  • define custom global methods to call alert()
  • use regexp or a "stop-word" to call alert()
  • use allow/block-lists or enable it for localhost only
  • use it to see console.log() events called in an iframe

The extension provides 3 modes to use.

You can switch the mode depending on your needs and the web-site's configurations
Default mode
[+] provides all the features
[+] injects script tag onto the page to extend console and window objects
[+] can show both alert and toast: simultaneously or separately depending on the called method
[-] requires page reload to apply any options' update
[-] will not work an sites with a good CSP(Content Security Policy) configuration
[-] may "loose" some events on fast, small or cached website due to the asynchronous nature of JavaScript
Toasts Only

[+] does not inject script tag, but instead uses browser's API only
[+] works on any website and is not affected by CSP(Content Security Policy)
[+] can not "loose" events but will show them with a short delay
[+] You do not need to reload the page to pause this extension or to apply any options' update
[-] can not extend console or window object to add custom or to modify existent methods
Simple mode
simple as a pay:
[+] injects script tag onto the page to add alert() method to the console object
[+] most likely will not "loose" events on fast, small or cached website as it's just a one line of JavaScript code
[-] implements only one single feature - maps alert() to console.alert() method
[-] requires page reload to pause it, but in most cases you do not need to pause it at all
[-] will not work an sites with a good CSP(Content Security Policy) configuration
